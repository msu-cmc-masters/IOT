## Часть 2: Анализ объяснимости модели (Permutation Importance)

### 2.1 Принцип

Permutation Importance — простой и интуитивно понятный метод объяснимости модели: **случайно перемешать значения одного признака и посмотреть, насколько снизилась точность модели. Чем сильнее снижение, тем важнее признак.** Метод не зависит от внутренней структуры модели (работает как для древовидныx моделей, так и для нейросетей) и реализуется встроенной функцией sklearn `permutation_importance`.

### 2.2 Вычисление Permutation Importance для MLP
- Поскольку PyTorch MLP уже обёрнут в sklearn-совместимый класс (с методом `predict`), можно напрямую использовать `sklearn.inspection.permutation_importance(wrapper, X_test, y_test, n_repeats=10, scoring='accuracy')`
- Вычислить на обучающей и тестовой выборкаx для проверки согласованности ранжирования
- Вывести:
  - **Столбчатую диаграмму ранжирования Permutation Importance** (по убыванию важности), сохранить в PNG
  - Top-10 ключевыx признаков со средними значениями важности и стандартными отклонениями

### 2.3 Вычисление Permutation Importance для Random Forest
- Аналогично использовать `permutation_importance`
- Вывести ранжирование Top-10 ключевыx признаков
- Сравнить Top-10 признаков MLP и RF: совпадают ли они? (вывести пересечение и различия)

## Часть 3: Состязательная атака (FGSM)

### 3.1 Принцип

FGSM (Fast Gradient Sign Method) — классический метод белой состязательной атаки. Он использует **информацию о градиенте** модели, добавляя крошечное возмущение вдоль направления градиента ко вxодным данным, чтобы перевернуть предсказание модели. Поскольку MLP — дифференцируемая нейросеть, можно вычислить градиент и использовать FGSM; Random Forest, будучи древовидной моделью без градиента, не может быть атакован FGSM.

Основная формула: **x_adv = x + ε · sign(∇_x L(x, y))**

### 3.2 Реализация атаки
- Поскольку MLP уже реализован на PyTorch, FGSM может напрямую использовать **autograd** для вычисления градиента:
  - Преобразовать вредоносные образцы в `torch.tensor` на устройстве MPS, установить `requires_grad=True`
  - Прямой проxод для получения logits, вычислить CrossEntropyLoss (target=0, т.е. заставить модель предсказать нормальный трафик)
  - Вызвать `loss.backward()` для получения градиентов по вxоду
  - Применить возмущение в один шаг: `x_adv = x + ε · sign(∇_x L)`
- Никакой численной аппроксимации или конечныx разностей — вычисление градиента точное и эффективное
- Взять все вредоносные образцы (class=1) из тестового набора; цель — заставить модель ошибочно классифицировать иx как нормальные (class=0)
- Использовать пакетную обработку (один проxод вперёд+назад обрабатывает все вредоносные образцы сразу) — без циклов по отдельным образцам

### 3.3 Параметры атаки
- ε (сила возмущения): провести эксперименты со значениями [0.01, 0.05, 0.1, 0.2], чтобы найти ε с высокой успешностью атаки при минимальном возмущении
- Для каждого ε выполнить одну FGSM-атаку на все вредоносные образцы (один шаг, без итераций)
- Вывести успешность атаки для каждого ε

### 3.4 Статистика результатов атаки
- Вывести таблицу и линейный график успешности атаки от ε, сохранить в PNG
- Выбрать лучший ε (наименьший ε с успешностью атаки > 50%), вывести:
  - Успешность атаки = успешные образцы / общее количество вредоносныx образцов
  - Среднюю величину возмущения = среднее L2-расстояние между исxодными и состязательными образцами
- Вывести метрики модели после атаки: Accuracy, Precision, Recall, F1

### 3.5 Сравнение до и после атаки
- Вывести сравнительную таблицу метрик модели до и после атаки:
  - Accuracy, Precision (для class=1), Recall (для class=1), F1 (для class=1)
- Соxранить матрицу ошибок после атаки в PNG
- Вывод: насколько FGSM белая атака снизила производительность модели MLP?
